# **GelSight Data Platform Modernization – One Page Summary**

## **Project Goal**

Build a governed, scalable Azure-based data platform that ingests GelSight’s scan outputs (images, STL/TMD/X3P files, and YAML analysis files), standardizes and models them, and enables automated analytics, quality reporting, and eventually machine learning–powered defect and surface‑feature detection. ## **Why This Project Exists**

GelSight’s devices generate high‑resolution surface scans used for quality inspection. Their current analysis workflow requires **manual operator steps** because the software is **not yet fully automated**. The outputs are stored in nested folders, vary by analysis routine, and lack a centralized data model.    [\[Data file...(1-9-2026) | OneNote\]](https://netrixonline.sharepoint.com/sites/EXT-191132GelSightInc/_layouts/15/Doc.aspx?action=edit&mobileredirect=true&wdorigin=Sharepoint&DefaultItemOpen=1&sourcedoc={feaa6735-e52e-43d8-85ac-559cf152b46d}&wd=target(/Meeting Notes.one/)&wdpartid={b8a351f4-e5e7-429f-be07-8fa8fc3f640d}{1}&wdsectionfileid={08739f8d-328f-4583-8b6b-79f9b64cf196})

The company wants to:

*   Eliminate manual analysis steps long-term
*   Centralize all scan data
*   Normalize the many routine‑specific YAML formats
*   Enable reliable reporting for manufacturing QA
*   Prepare the data to train ML models and deploy them down to the edge    [\[Summary Explanation | OneNote\]](https://netrixonline.sharepoint.com/sites/EXT-191132GelSightInc/_layouts/15/Doc.aspx?action=edit&mobileredirect=true&wdorigin=Sharepoint&DefaultItemOpen=1&sourcedoc={feaa6735-e52e-43d8-85ac-559cf152b46d}&wd=target(/File Ingestion.one/)&wdpartid={00212606-b11e-05ed-01cc-02f4613db4fe}{1}&wdsectionfileid={6348dfa9-e5c8-4beb-86c5-e27d900da61d}), [\[Data file...(1-9-2026) | OneNote\]](https://netrixonline.sharepoint.com/sites/EXT-191132GelSightInc/_layouts/15/Doc.aspx?action=edit&mobileredirect=true&wdorigin=Sharepoint&DefaultItemOpen=1&sourcedoc={feaa6735-e52e-43d8-85ac-559cf152b46d}&wd=target(/Meeting Notes.one/)&wdpartid={b8a351f4-e5e7-429f-be07-8fa8fc3f640d}{1}&wdsectionfileid={08739f8d-328f-4583-8b6b-79f9b64cf196})

## **What We Are Building**

### **1. Data Ingestion (Bronze Layer)**

Extract and ingest all artifacts from customer‑provided ZIP drops:

*   Raw surface images
*   3D surface files (STL, TMD, X3P)
*   YAML `scancontext` analysis files
*   Subfolders such as `analysis/` for each routine    [\[Data file...(1-9-2026) | OneNote\]](https://netrixonline.sharepoint.com/sites/EXT-191132GelSightInc/_layouts/15/Doc.aspx?action=edit&mobileredirect=true&wdorigin=Sharepoint&DefaultItemOpen=1&sourcedoc={feaa6735-e52e-43d8-85ac-559cf152b46d}&wd=target(/Meeting Notes.one/)&wdpartid={b8a351f4-e5e7-429f-be07-8fa8fc3f640d}{1}&wdsectionfileid={08739f8d-328f-4583-8b6b-79f9b64cf196})

We also create logic to:

*   Link images + shapes + routine outputs
*   Preserve folder structure and metadata
*   Prepare data for normalization

***

### **2. Data Transformation & Modeling (Silver Layer)**

Because GelSight has “tens” of analysis formats (Offset, HoleDiameter, DefectDetection, PitDetection, Roughness, etc.), we design a flexible schema that supports variability.    [\[1/9/2026 F...n Overview | OneNote\]](https://netrixonline-my.sharepoint.com/personal/amorgan_netrixllc_com/_layouts/15/Doc.aspx?action=edit&mobileredirect=true&wdorigin=Sharepoint&DefaultItemOpen=1&sourcedoc={d70b24d5-67a6-486a-9ebd-6b219fd338e4}&wd=target(/Customers.one/)&wdpartid={af5519f3-7047-473b-b3ee-710bcccb7775}{1}&wdsectionfileid={27ced749-1e15-4e45-84c1-bbd572ce27c6})

Key modeling decisions:

*   **Shapes → Dimension tables** (circle, line, rectangle, ruler, etc.)
*   **Routines → Fact tables** (offset measurements, hole diameters, defects, roughness metrics)
*   **Scan metadata → Context tables**

This structure enables BI, trending, and ML training.

***

### **3. Analytics, QA Reporting, and ML‑readiness (Gold Layer)**

Produce clean, consistent, analytics‑ready tables for:

*   Quality inspection reporting
*   Surface‑feature measurement trends
*   Scan pass/fail analysis
*   Operator performance evaluation
*   Dataset preparation for computer‑vision and ML pipelines    [\[Summary Explanation | OneNote\]](https://netrixonline.sharepoint.com/sites/EXT-191132GelSightInc/_layouts/15/Doc.aspx?action=edit&mobileredirect=true&wdorigin=Sharepoint&DefaultItemOpen=1&sourcedoc={feaa6735-e52e-43d8-85ac-559cf152b46d}&wd=target(/File Ingestion.one/)&wdpartid={00212606-b11e-05ed-01cc-02f4613db4fe}{1}&wdsectionfileid={6348dfa9-e5c8-4beb-86c5-e27d900da61d})

This also supports GelSight’s long‑term goal:  
**automating the currently manual inspection steps using ML models and pushing those models back to the device/edge.**    [\[Data file...(1-9-2026) | OneNote\]](https://netrixonline.sharepoint.com/sites/EXT-191132GelSightInc/_layouts/15/Doc.aspx?action=edit&mobileredirect=true&wdorigin=Sharepoint&DefaultItemOpen=1&sourcedoc={feaa6735-e52e-43d8-85ac-559cf152b46d}&wd=target(/Meeting Notes.one/)&wdpartid={b8a351f4-e5e7-429f-be07-8fa8fc3f640d}{1}&wdsectionfileid={08739f8d-328f-4583-8b6b-79f9b64cf196})

*   The device software produces YAML files describing “shapes” (geometry) and “routines” (measurements). ## **Future State Vision**

A fully automated, cloud‑driven platform where:

1.  Scans are uploaded automatically
2.  Analysis runs without human intervention
3.  Measurements are standardized and versioned
4.  Analytics dashboards refresh in real‑time
5.  ML models detect defects or surface features
6.  The models are deployed back to GelSight's hardware for autonomous inspection

This project creates the **foundational data platform** GelSight needs to modernize their inspection workflow, reduce manual labor, and enable AI‑driven surface analysis at scale.
